TA Management System

Our capstone course had us create a TA management system that was to replace the current system of TA Management used by UBCO. The client (the professor at UBCO doing the allocations) was providing feedback and selected our project as the most suitable to start being prepared for use. I had the opportunity to continue this work over 2 independent studies, and have made significant improvements to the project in aiding its preparation for production use.

The project uses SpringBoot for the backend RESTful API, which is Java based, and Vite with React on the frontend. A large amount of work went into refactoring from microservices, which we implemented in the capstone, back to a modular monolithic application for easier maintenance. This led to major changes in the documentation, database, tests, and overall backend implementation. I then focused on new features focused on easier application processes and assignment of TAships.

Below shows an example workflow of our system.

First, a student can create an application for a given semester that is active, specifying information like how many hours they would be willing to work and their course department preferences.

Now, the coordinator can view all the courses and applications for a semester, with their relevant time slots to try and allocate students based on their current hours allocated and availability.

Once the offer is sent, the student can view the work assignment details and either accept or decline.

If the student accepts or declines, this updates in the coordinators view.
